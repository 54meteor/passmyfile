# 隔空投送 - 移动端扩展方案

## 当前架构
- 技术栈：Tauri 2.x + React + TypeScript + Rust
- 传输协议：Rust TCP（端口 18766），自定义二进制协议
- 状态：Windows ✅ 完成，Android ⏳ 待打包

## 手机端扩展讨论

### 背景
- 主公希望用 UniApp 封装手机端
- 但 Rust 使用的是 TCP 二进制协议，UniApp 无法直接调用

### 待选方案

#### 方案一：Rust 加 HTTP 层（推荐）
在 src-tauri 里加一个轻量 HTTP 服务（axum 或 actix-web），暴露 REST API，UniApp 调 API 即可。

```
手机(UniApp) → HTTP API (Rust) → 复用现有传输逻辑
```

#### 方案二：独立 Go 服务
另外起一个 Go HTTP 服务，复用传输核心逻辑，手机端调 Go API。

```
手机(UniApp) → Go HTTP API → 传输文件
```

### 结论
待定，暂不实施。先继续推进餐饮点餐系统桌面端开发。
